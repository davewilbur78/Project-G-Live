DECISIONS MADE THIS SESSION

TIMESTAMP: 2026-05-08 21:36 UTC -- The session declared itself
an EXPLORE session. The first task was to evaluate the inherited
six-mode system (BUILD, DEBUG, PLAN, MODIFY, REVIEW, TEST) from
Project-G. Decision: question everything rather than carry it
forward.

TIMESTAMP: 2026-05-08 21:39 UTC -- Evaluated each of the six
modes critically. Found DEBUG and MODIFY redundant with BUILD.
Found REVIEW too rigid (can't fix something found broken). Found
TEST a subset of REVIEW. Found PLAN useful but too narrow. Found
BUILD solid. Decision: redesign from scratch.

TIMESTAMP: 2026-05-08 21:43 UTC -- User clarified the origin of
the mode system: it was a practical guard against a specific
failure pattern -- AI sessions defaulting to BUILD posture at
boot even when the session was meant to fix something, compounding
problems. The mode declaration was a forcing function to make the
AI stop and ask. This reframed the entire design problem.

TIMESTAMP: 2026-05-08 21:46 UTC -- Three postures chosen to
replace six modes: BUILD / FIX / EXPLORE. Rationale: broad enough
to allow natural work flow, distinct enough to communicate intent,
designed around the actual failure pattern being guarded against.

TIMESTAMP: 2026-05-08 21:46 UTC -- Posture transitions within a
session explicitly permitted. Transitions do not require new
threads. Continuity is the point. Declaration + TIMESTAMP is
sufficient. Forcing new threads to switch posture breaks the
context window and loses work.

TIMESTAMP: 2026-05-08 21:51 UTC -- Context window identified as
the real constraint, not posture. The system must monitor context
health actively. 60% checkpoint: write snapshot, commit, continue.
80% checkpoint: write snapshot, commit, ask user. Critical: execute
distress protocol immediately without waiting for signal.

TIMESTAMP: 2026-05-08 22:01 UTC -- Session snapshot architecture
designed. Key decisions: snapshots named by datetime
(SESSION-YYYY-MM-DD-HHMM-UTC.md), committed to /sessions/,
never deleted, never overwritten. The archive is the project's
mind, not just its history.

TIMESTAMP: 2026-05-08 22:05 UTC -- Transcripts: on demand, not
by default. Triggered by signal phrase. Full reconstruction
required, not summary. Summaries are lossy and lose the nuance
that makes transcripts valuable in the first place.

TIMESTAMP: 2026-05-08 22:11 UTC -- Signal vocabulary designed.
Four categories: exit signals (preserve and close), distress
signals (freeze frame, capture, stop forward work), health checks
(report context status), transcript signals (full reconstruction
and commit). Execute before responding. Always.

TIMESTAMP: 2026-05-08 22:17 UTC -- Distress signal protocol
refined: "stop all forward work" must be preceded by naming
exactly what was in motion. Do not let the frame drop. Capture
first, then stop. The order matters.

TIMESTAMP: 2026-05-08 22:21 UTC -- Restoration prompt format
designed. Required elements: hard facts with TIMESTAMPs, decision
trail chronological, open threads specific, partially built work
(complete not summary), DO NOT DO THIS list, single next action,
confirmation gate. AI does not proceed until user confirms.

TIMESTAMP: 2026-05-08 22:25 UTC -- "Paste fragments" language

Replaced with: fully reconstruct the work so a future session

to wip/ branch and reference. Full reconstruction in snapshot
only as fallback.
